# 표 2. RAG/Vector DB 설계 요약(1차년도 구조 설계 증빙, 부록 B-5 기반)

아래 표는 HWP에 **표로 붙여넣기** 용도로 정리한 요약본입니다.

| 구분 | 설계 내용(요약) |
|---|---|
| 목표 | SHAP(모델 해석) + RAG(문서 근거) 결합으로 **근거 제시형 설명** 제공, 설명 일관성/운영성 확보 |
| 지식 소스(예) | 교육/도메인 문서, 서비스/콘텐츠 메타(문항/개념), 설명 가이드(SHAP 해석 가이드), FAQ/정책/용어집 |
| 인덱싱(예) | 문단/항목 단위(약 300~800자) chunk, 메타데이터(`source_type`, `doc_id`, `concept_id`, `grade`, `version`, `updated_at`) 부여 |
| 검색 전략(예) | Top-k 3~5, 메타 필터(학년/과목/개념/콘텐츠 타입) 우선, (선택) 재랭킹은 2차년도 적용 |
| 근거 제시 규칙 | 응답에 **SHAP Top feature 근거 + RAG citations(출처/스니펫)** 동시 포함, 출처 없는 단정 금지 |
| 안전장치 | 환각 방지(근거 부족 시 “확인 필요/추가 정보 요청”), 민감정보 노출 제한 |
| 품질 평가(2차년도 연계) | 근거 포함률, 검색 품질(hit@k/MRR), 전문가 루브릭 또는 Q&A set 기반 정답성/일관성 평가 |
